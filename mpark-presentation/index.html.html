Google Apps Script 프로젝트

데이터 추출, 자동화

매주 반복되던 수동 데이터 추출 및 보고서 작성 업무를 자동화하여,
반복업무 시간을 단축하고 데이터 신뢰도를 향상시키는 어플리케이션 구축.

문제점 (As-Is)

  • ✘
    반복적 시간 소요: 단순 반복 작업에 매주 시간 소요
  • ✘
    인적 오류 발생: 데이터 누락, 오기입 등 실수 발생 가능성 상존
  • ✘
    업무 비효율성: 담당자 부재 시 업무 지연 및 데이터 확인의 어려움

개선 방안 (To-Be)

  • ✔
    완벽 자동화: 스케줄러를 통해 매주 데이터 자동 추출 및 시트 생성
  • ✔
    데이터 정확성 100%: 시스템 API 연동으로 인적 오류 원천 차단
  • ✔
    사용자 편의성: 필요시 원하는 기간을 직접 선택하여 수동 실행 가능

자동화 도입을 통해 조직 전체의 업무 효율성을 높일 수 있습니다.

매주 데이터 추출 및 Sheet 작성, 합계/평균 계산등의 반복 작업 개선.

📈 데이터 신뢰도 향상

시스템 자동화로 인적 오류 해결, 데이터 신뢰도 향상.

📈 업무 유연성 확보

필요시 비정기 데이터도 즉시 추출 및 활용 가능하여 담당자 부재시 유연한 대응.

📈 낮은 개발 & 유지보수 난이도

javascript 기반 낮은 개발 난이도로 개발 및 유지보수 용이.

💡 기술 상세 정보

본 자동화 시스템을 구현하고 운영하는 데 필요한 핵심 기술 설정 정보입니다.

Google Apps Script 권한 (OAuth Scopes)

스크립트가 정상 동작하기 위해 사용자로부터 다음 Google 서비스 접근 권한 승인이 필요합니다.

  • script.external_request: 외부 API 서버와 통신
  • spreadsheets: Google Sheets 읽기, 생성, 쓰기
  • script.send_mail: 사용자 계정으로 메일 발송
  • script.container.ui: 커스텀 메뉴 및 팝업창 표시

UrlFetchApp 주요 옵션

API 서버와 안정적으로 통신하기 위한 핵심 옵션입니다.

const options = {
  'method': 'post',
  'contentType': 'application/json',
  'muteHttpExceptions': true,
  'validateHttpsCertificates': false,
};
  • muteHttpExceptions: true - API 에러 발생 시 스크립트가 중단되지 않고 에러 코드를 직접 처리할 수 있게 합니다.
  • validateHttpsCertificates: false - 내부망의 사설 HTTPS 인증서 사용 시 유효성 검사를 건너뜁니다.
    *보안 주의* (개발 서버 통신시 필요)

ALB 요청사항

  • 1. 외부 도메인 등록 (ex. https://api-pulse.hanjin.com)
  • 2. 외부(internet-facing) ALB 생성
  • 3. ALB - EC2 (ex. ec2-lg-pulse-prd-osrm) 연결

API 서버 주요 설정

  • 엔드포인트: 외부 도메인 URL 및 POST 메소드 지원
  • 인증서: SSL 인증서 등록 필요
  • (중요) 방화벽 설정: Google 서버 IP 대역 전체에 대한 인바운드(Inbound) 규칙 허용
  • (중요) ALB 지역 제한: KR & US 모두 접근 허용
